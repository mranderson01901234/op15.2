# Git Workflow Guide - Keeping Branches Synced

## Quick Workflow

After making commits and pushing, run:

```bash
./scripts/sync-branches.sh
```

This script will:
1. ✅ Push `op15.2` to both `op15.2` and `origin` remotes
2. ✅ Merge `op15.2` into `op15.2/main` automatically
3. ✅ Keep everything synced

## Manual Workflow (if script doesn't work)

### Step 1: Push your commits
```bash
git push op15.2 op15.2        # Push to op15.2 repository
git push origin op15.2        # Push to origin repository
```

### Step 2: Merge into main
```bash
git checkout -b temp-merge op15.2/main
git merge op15.2/op15.2 -m "Merge op15.2 into main"
git push op15.2 temp-merge:main
git checkout op15.2
git branch -D temp-merge
```

## Recommended: Set up branch tracking

To make pushing easier, set up tracking:

```bash
# Track op15.2 remote for op15.2 branch
git branch --set-upstream-to=op15.2/op15.2 op15.2

# Now you can just use:
git push  # Automatically pushes to op15.2/op15.2
```

## Best Practices

1. **Always work on `op15.2` branch**
2. **After committing**: Run `./scripts/sync-branches.sh`
3. **Before starting new work**: Run `git pull op15.2 op15.2` to get latest
4. **Check status**: `git status` before syncing

## Troubleshooting

### Script fails with "uncommitted changes"
```bash
git stash              # Save changes
./scripts/sync-branches.sh
git stash pop          # Restore changes
```

### Script fails with merge conflicts
```bash
# Resolve conflicts manually, then:
git add .
git commit -m "Resolve merge conflicts"
git push op15.2 temp-merge:main
```

### Want to see what would be merged?
```bash
git log op15.2/main..op15.2/op15.2 --oneline
```
